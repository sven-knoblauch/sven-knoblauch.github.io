Perceptron Learn Algorithm

Let's first discuss a simple perceptron and then inspect a simple dataset, on which we will train the perceptron. A Perceptron is a neuron with adaptable weights \(w\) and and a threshold \(\theta\). The class labels are \(1\) or \(-1\). The test to classify a point \(x\) is described with the dot product \(y = \langle w, x\rangle - \theta \). If the result is \(y \geq 0\) then the predicted class is \(1\), otherwise the class is \(-1\). For the Algorithm, for each step, it will be interated over all points, and when the predicted class missmatches the class label, a \( weights \) is computed, which changes the weight vector and with that also the seperation line to have a better decision. To make the calculations more easier, the bias \(\theta \) will be concatenated with the weight vector and for each point, one dimension will be added with a constant of 1.
  • for each point in the data do
    • calculate \(y = dot(weights, point) * class\)
    • if \(y < 0\) then point is wrongly classified
      • calculate \( \Delta x = weights * class \)
      • calculate new \( weights = weights + \Delta x\)
  • repreat until all points are correctly classified or maximum steps is reached

Convergence

If the problem is linear saparable, the algorithm stops after a finite number of steps and finds a decision line, which seperates all given points correcly. If the problem is not linear separable, kernel function can be used to project the data in higher dimensional spaces, where the data can then be linear separated. A drawback of the perceptron algorithm is, that the decision line has not a maximum margin, which can lead to bad results in some cases, where unseen data gets wrongly classified. To counter this, a support vector machine can be used, which maximizes also the margin.
